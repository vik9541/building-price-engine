├─ Производитель не найден

### 4.2 АЛГОРИТМ ОПРЕДЕЛЕНИЯ УРОВНЯ НАДЁЖНОСТИ

```python
def calculate_confidence_level(product):
    """
    1/2/3 уровень доверия в зависимости от:
    - Привязки к производителю
    - Количества источников
    - Консистентности данных
    """
    
    score = 0
    
    # УРОВЕНЬ 1: Производитель найден и проверен
    if has_official_manufacturer:
        if has_manufacturer_sku and verified_official:
            score += 50  # Максимум за производителя
        elif has_manufacturer_sku:
            score += 30
        else:
            score += 10
    
    # УРОВЕНЬ 2: Маркетплейсы
    marketplace_sources = count_reliable_sources()  # 3+?
    if marketplace_sources >= 5:
        score += 40
    elif marketplace_sources >= 3:
        score += 25
    elif marketplace_sources >= 1:
        score += 10
    
    # УРОВЕНЬ 3: Аномалии/красные флаги
    if price_anomaly or single_source or no_manufacturer:
        score -= 30
    
    # ИТОГОВЫЙ УРОВЕНЬ
    if score >= 70:
        return LEVEL_1  # Привязано к производителю ✅✅✅
    elif score >= 40:
        return LEVEL_2  # Маркетплейсы 🔶
    else:
        return LEVEL_3  # Сомнительно ⚠️
```

## ⚠️ ПРОБЛЕМЫ И РЕШЕНИЯ

### ПРОБЛЕМА 1: "Товар найден только в 1 источнике"

```
РЕШЕНИЕ:
├─ Пометить как LEVEL_3 (сомнительный)
├─ Поискать через Google (site:производитель.ru "название товара")
├─ Проверить, не снят ли с производства
├─ Если не найдено → поместить в SUSPICIOUS_PRODUCTS

ДЕЙСТВИЕ:
├─ Попытка найти реального производителя
├─ Если не нашли → помечаем как "possible_fake"
└─ Требуется верификация перед продажей
```

### ПРОБЛЕМА 2: "Производитель не ответил на запрос"

```
РЕШЕНИЕ:
├─ Использовать информацию из маркетплейсов (3+ источника)
├─ Если совпадение >90% → считаем доверяемым
├─ Помечаем как LEVEL_2 (маркетплейсы)
└─ Со временем пытаемся снова связаться

ТАЙМАУТ:
└─ Повторный контакт через 30 дней
```

### ПРОБЛЕМА 3: "Каталожный номер не совпадает"

```
РЕШЕНИЕ:
├─ Сравниваем характеристики, а не только номер
├─ Если характеристики совпадают на 95% → считаем одним товаром
├─ Если не совпадают → отдельная запись

ПРИМЕР:
├─ У нас: "Кирпич красный 250x120x65"
├─ У Петровича: "SKU: 12345, Кирпич красный"
├─ У производителя: "Каталог: BR-250-Red"
└─ Делаем соответствие по характеристикам, а не по SKU
```

### ПРОБЛЕМА 4: "Цена в маркетплейсах сильно отличается"

```
РЕШЕНИЕ:
├─ Если >30% разница → помечаем как аномалию
├─ Проверяем минимальный заказ
├─ Проверяем наличие опций (доставка, монтаж)
├─ Если это опции → объясняем разницу
├─ Если действительно разница → оповещаем модератора

FLAG: "price_inconsistency_high"
```
